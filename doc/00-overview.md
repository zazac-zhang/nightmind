# NightMind 产品白皮书

> 版本: V1.0 | 日期: 2023-10 | 核心理念: Close your eyes, open your mind.

---

## 1. 产品愿景与定位

### 1.1 产品定义

NightMind 是一款**睡前认知巩固应用**。它利用神经科学中的"睡眠依赖型记忆巩固"（Sleep-dependent Memory Consolidation）原理，在用户睡前 1 小时的黄金窗口期，通过深度大模型 Agent 进行知识总结、间隔复习、关联构建与深度思考引导。

### 1.2 核心价值

| 对象 | 价值 |
|------|------|
| **对用户** | 将碎片化知识转化为长期记忆，利用睡眠时间"后台加工"知识，无需额外占用白天精力 |
| **对生态** | 作为现有学习工具（Anki, Obsidian, Notion）的**智能调度层**，不替代记录，只增强记忆与理解 |
| **技术定位** | 深度 Agent 化应用。不仅是工具，更是具备记忆、规划、情感感知能力的"夜间导师" |

### 1.3 目标场景

| 维度 | 描述 |
|------|------|
| **时间** | 睡前 30-60 分钟 |
| **环境** | 昏暗、安静、躺卧状态 |
| **状态** | 身体疲劳但思维活跃，适合深度联想，不适合高强度操作 |

## 2. 核心设计哲学

### 2.1 零交互原则 (Zero-UI Philosophy)

- **零视觉**: 所有信息通过语音（TTS）传递，屏幕仅作为状态指示，用户无需阅读文字
- **零手动**: 全程语音控制 + 触觉反馈。用户双手可被占用（如抱枕头、戴眼罩），无需点击、滑动或输入
- **零摩擦**: 启动无需解锁/找 App，结束无需确认/保存。技术细节完全隐形

### 2.2 认知优先原则 (Cognition-First)

- **脑力最大化**: 用户的认知资源应全部用于"思考问题"，而非"操作软件"
- **负荷自适应**: Agent 实时感知用户疲劳度，动态调整内容深度与交互频率
- **睡眠保护**: 学习服务于睡眠，而非牺牲睡眠。会话结束必须以心理放松为终点

### 2.3 内容转化原则 (Content Transformation)

- **意象优于定义**: 拒绝朗读定义，优先使用比喻、类比、故事
- **思想优于代码**: 拒绝朗读代码/公式，转为解释设计意图与核心逻辑
- **简洁不失真**: 语言极简，但保留知识的核心骨架与边界

## 3. 交互体验设计

### 3.1 启动与退出 (Entry & Exit)

#### 无感启动

| 方式 | 描述 |
|------|------|
| **自动化** | 检测到充电 + 连接卧室 WiFi + 光线变暗 → 震动提示 → 语音询问"准备好回顾今天了吗？" |
| **物理触发** | 耳机双击 / 床头 NFC 触碰 |
| **手动触发** | 桌面唯一大按钮，点击即入语音模式 |

#### 自然休眠

```
用户说"晚安"或检测到入睡（呼吸平稳/无响应）
  ↓
播放渐弱白噪音
  ↓
自动保存进度
  ↓
进入休眠监听
```

### 3.2 语音交互流 (Voice Flow)

#### 全双工通信 (Full-Duplex)

- 支持随时打断 (Barge-in)。用户无需等待 Agent 说完
- VAD (语音活动检测) 区分"思考停顿"与"说完停顿"。思考时保持静默陪伴，说完后自动响应

#### 触觉反馈语言 (Haptic Language)

| 模式 | 含义 |
|------|------|
| 短震 1 次 | 回答质量高/认可 |
| 短震 2 次 | 回答有偏差/温和纠正 |
| 长震 1 次 | 话题切换/新知识点 |
| 轻颤 | Agent 正在检索/思考中 |

> **原则**: 触觉仅辅助，核心信息必通过语音传达

### 3.3 中断与恢复机制 (Interruption & Recovery)

这是体验的核心壁垒，需实现"无缝衔接"。

#### 状态快照 (State Snapshot)

- Agent 每 30 秒或每个语义单元结束时，保存上下文快照：
  - 当前话题
  - 最后锚点
  - 下一步计划
  - 用户状态

#### 话题栈管理 (Topic Stack)

- 支持多层级话题切换（A→B→C）
- 恢复时按栈顺序返回（C→B→A）
- 限制栈深度（最多 3 层），防止思维迷失

#### 恢复话术 (Recovery Script)

```
公式: [话题锚点] + [最后位置] + [自然过渡]

示例: "我们刚才聊到**装饰器的嵌套**（锚点），
       正想请你**举个生活中的例子**（位置）。
       **还要继续吗？**（过渡/确认）"
```

#### 异常处理

| 场景 | 处理 |
|------|------|
| 网络中断 | 切换本地轻量模型，诚实告知，恢复后无缝衔接 |
| 外部干扰 | 自动暂停占位（低音量白噪音），用户返回后轻声唤醒 |

## 4. 内容生成策略

### 4.1 1 小时会话节奏 (Session Rhythm)

| 时间段 | 阶段 | 认知负荷 | 内容策略 |
|--------|------|----------|----------|
| 00-05m | 暖场校准 | 低 | 状态确认、精力评估、今日概览 |
| 05-25m | 核心深度区 | 高 | 1-2 个核心概念深度对话、苏格拉底追问 |
| 25-45m | 关联巩固区 | 中 | 间隔复习、知识网络连接、旧知回顾 |
| 45-55m | 种子沉淀区 | 低 | 提出"种子问题"供潜意识加工、故事化总结 |
| 55-60m | 温柔收尾 | 极低 | 肯定、放下、白噪音过渡、助眠 |

### 4.2 知识类型转换规则

LLM 必须对原始知识进行"语音友好化"处理：

#### 代码/技术

- ❌ `def decorator(func): ...`
- ✅ "装饰器本质是'函数包装器'，就像给礼物加包装纸，不改变礼物本身，但增加了额外功能。"

#### 数学/公式

- ❌ `P(A|B) = ...`
- ✅ "贝叶斯本质是'用新证据更新旧信念'。比如看到乌云，更新下雨的概率。"

#### 复杂概念 (如燕尾定理)

- ❌ 定义 + 证明 + 变体
- ✅ "核心是'面积比例传递'。想象一个 Y 字形，两边面积比等于底边比，像杠杆原理。记住'Y 形传递'这个画面。"

#### 列表/对比

- ❌ 第一点、第二点...
- ✅ 分组隐喻："这三点可以记成'种子 - 生长 - 结果'的生命周期。"

### 4.3 内容校验清单

LLM 生成内容前需自我校验：

- [ ] 能否在 15 秒内朗读完？
- [ ] 是否包含必须"看"才能理解的内容？
- [ ] 是否有一个具体的意象帮助想象？
- [ ] 是否给用户留出了思考间隙？

## 5. 生态集成策略

NightMind 不做数据孤岛，做**智能增强层**。

### 5.1 对接 Anki (双向同步)

| 方向 | 功能 |
|------|------|
| **读** | 拉取今日需复习卡片 → Agent 转化为对话问题 → 复习结果回写 Anki 间隔 |
| **写** | 睡前对话产生的新理解 → 自动生成 Anki 卡片 → 推送到 Anki 牌组 |

### 5.2 对接笔记工具 (Obsidian/Notion)

| 模式 | 功能 |
|------|------|
| **被动摄入** | 监听特定文件夹/Tag → 自动分块、摘要、向量化 |
| **主动回写** | 睡前总结生成 Markdown → 写入每日日志 (Daily Note) |
| **双向链接** | Agent 发现笔记间关联 → 自动在 Obsidian 中插入 [[链接]] |

### 5.3 对接 Readwise/Reader

- 自动同步高亮内容 → Agent 进行"费曼技巧"处理 → 睡前让用户复述高亮含义

## 6. 隐私与伦理

| 原则 | 说明 |
|------|------|
| **数据主权** | 支持本地向量库与本地模型 (Ollama) 部署，敏感数据不出内网 |
| **隐私过滤** | STT 前本地正则过滤敏感词（密码、隐私日记） |
| **睡眠优先** | 检测到用户过度使用或情绪焦虑时，Agent 主动建议停止，保护睡眠健康 |
| **透明性** | 明确告知用户 AI 生成内容的局限性，不作为医疗或专业建议 |

---

## 7. 成功标准

我们的成功标准不是日活或时长，而是：

> **用户摘下耳机时，是否感觉"像和一个懂你的朋友深夜聊了会儿天"，并且带着对知识的新洞察，安然入梦。**
